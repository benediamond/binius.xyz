# AWS Benchmarks Guide [How to Reproduce Binius Benchmark Results in AWS]

## Getting Started with AWS EC2

Follow this [AWS tutorial](https://aws.amazon.com/ec2/instance-types/) to launch an EC2 instance - ensure you are running Linux OS, e.g. `Ubuntu Server 24.04 LTS`. We recommend running `c7i.16xlarge` (x86, Intel) and `c8g.16xlarge` (arm, Graviton) instances.

## Running Binius Examples on AWS EC2 Instances

Once done with setting up the AWS instance, update linux kerel:

```bash
sudo apt update && sudo apt upgrade -y
sudo reboot
```

You will have to `ssh` to your AWS instance again. Next step is to install the `rust` toolchain and to configure env variables:

```bash
sudo apt install build-essential
sudo apt install rustup
rustup default stable
export RUSTFLAGS="-C target-cpu=native"
rustc --print cfg -C target-cpu=native | grep gfni
```

This is a good time for a sanity checkpoint, prior to running the benchmars:

```bash
rustc --version && cargo --version
```

If you get something like this, that means that you've done all the steps correctly so far:

```bash
rustc 1.89.0 (29483883e 2025-08-04)
cargo 1.89.0 (c24e10642 2025-06-23)
```

Now it's time to clone the Binius repo:

```bash
git clone https://github.com/IrreducibleOSS/binius.git
```

Head to the `examples` folder:

```bash
cd binius/crates/examples/examples
```

Tip: Folder `/crates` contains `README.md` file which explains our cuircit philosophy.

We are going to run the `ethsign` example, feel free to try them all out.

```bash
cargo run --release --example ethsign --package binius-examples -- prove
```

Tip: Note that adding `--no-default-features` turns off the multithreading. We report all our benchmarks for boths single- and multi-threaded performance.
