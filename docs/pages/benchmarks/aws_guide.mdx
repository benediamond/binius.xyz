# AWS Benchmarks Guide [How to reproduce Binius64 benchmark results in AWS]

## Getting Started with AWS EC2

Follow this [AWS tutorial](https://aws.amazon.com/ec2/instance-types/) to launch an EC2 instance - ensure you are running Linux OS, e.g. `Ubuntu Server 24.04 LTS`. We recommend running `c7i.16xlarge` (`x86`, Intel) and `c8g.16xlarge` (`aarch64`, Graviton) instances.

## Running Binius Examples on AWS EC2 Instances

Once done with setting up the AWS instance, update linux kerel:

```bash
sudo apt update && sudo apt upgrade -y
sudo reboot
```

You will have to `ssh` to your AWS instance again. Next step is to install the `rust` toolchain and to configure env variables:

```bash
sudo apt install build-essential
sudo apt install rustup
rustup default stable
export RUSTFLAGS="-C target-cpu=native"
```

<Callout type="tip">
  Tip: This is a good time for a sanity checkpoint, prior to running the benchmars:
</Callout>

```bash
rustc --version && cargo --version
```

If you get something like this, that means that you've done all the steps correctly so far:

```bash
rustc 1.89.0 (29483883e 2025-08-04)
cargo 1.89.0 (c24e10642 2025-06-23)
```

Now it's time to clone the Binius64 repo:

```bash
git clone https://github.com/IrreducibleOSS/binius64.git
cd binius64
```

We are going to run the `ethsign` benchmark, feel free to try out [all.](https://github.com/IrreducibleOSS/binius64/tree/main/crates/examples/benches)

```bash
cargo run --release --example ethsign --package binius-examples -- prove
cargo bench --bench ethsign
```
