# The Shift Reduction [The technical core of Binius64]

The shift reduction is a key internal routine in Binius64. It is a common reduction target of both the [AND](/blueprint/backend/ands) and [MUL](/blueprint/backend/muls) reductions.

The shift reduction takes, as input, evaluation claims on [oblong-multilinearizations of constraint arrays](/blueprint/backend/mathematizing). The entire shift reduction is designed to process claims of this form, and in particular to reduce them to direct claims on the witness. To achieve this, we engage in a complex, multi-phase sumcheck. Making that sumcheck efficient for our prover takes work, as we explain below.

We carry out this program in the following sub-pages.
- [**The Sumchecks.**](sumchecks) Here, we show that the sumcheck the parties need to run can be naturally broken up into two phases. The idea is very vaguely Spartan-like [[Set20](https://eprint.iacr.org/2019/550)], but more complicated.
- [**Prover Implementation.**](implementation) Here, we precisely explain our _prover implementation_ of the above protocol. It's far from obvious how that protocol's prover should be implemented efficiently—and in particular, in a way that appropriately leverages the constraint system's sparsity.
