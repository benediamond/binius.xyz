# AND Constraints [Evaluating bitwise ANDs of 64-bit words]

To treat AND constraints, we need to do a number of cryptographic steps. First, we need to arithmetize the problem using multilinears. Next, we need to oblong-multilinearize everything, and run the univariate skip.

These steps serve to reduce our initial constraint-satisfaction problem to a lower-level one, involving evaluation claims on certain multilinears _related_ to the witness. Next, we need to further reduce those claims, until they become claims about the witness itself.

We explain these steps in detail throughout the below pages.

- [**The Shift Indicators.**](indicators) In this page, we prepare a key mathematical building block. These are polynomials whose restrictions to the cube "indicate" arithmetic offsets. In this effort, we wind up generalizing and building off of [[DP25](https://eprint.iacr.org/2023/1784), § 4.3].
- [**Oblong Multilinears.**](oblong) Here, we explain how we apply the univariate skip to the AND-constraint-satisfication problem. Along the way, we need to explain what _oblong multilinears_ and the _univariate skip_ are.
- [**The Shift Reduction.**](reduction) This step takes the _output_ of the univariate skip reduction and further reduces it, until it becomes a claim about the witness. It's a complicated, multi-phase sumcheck. We describe that sumcheck and some considerations around its implementation.
