# Blueprint [Cryptographic details about our backend]

In this site section, we explain the cryptographic backend of Binius64.

## Site Directory

We've split this site area into a few subsections.

<Callout type="note">
<Collapsible>
[**Math Background.**](math) This section contains a crash course on the mathematical objects we use.

Start here if you're new. We can't promise to cover everything, but it'll be a useful starting point.
</Collapsible>

<Callout type="note">
<Collapsible>
[**Constraint Systems.**](constraints) _Constraint systems_ are how we specify the computations we set about proving and verifying. Think somewhere in between "circuits" and "R1CS".

Binius64 introduces a new constraint system framework. Our constraint system basically takes R1CS, makes it operate fundamentally on 64-bit unsigned integers, and moreover enriches it by natively supporting constraints that act upon (logical, arithmetic, etc.) _bitshifts_ of this witness data. We also bake in support for unsigned, 64-bit integer multiplication.

The constraint system we get out is predictable and simple, but much more useful and efficient than plain, vanilla R1CS in characteristic 2.
</Collapsible>
</Callout>

<Callout type="note">
<Collapsible>
[**The Backend.**](backend) Here, we explain our entire cryptographic backend. We record separate protocols for checking the prover's [AND](constraints/ands) and [MUL](constraints/muls) constraints. These two are very different; the first uses a variant of Gruen's univariate skip, while the second uses GKR. Internally, both protocols reduce to a common internal reduction target, called the _[shift reduction](backend/reduction)_.

Along the way, we need to develop a key technical idea, called _[shift indicators](backend/shifts)_. These let us capture bitshifts in an algebraically meaningful way.
</Collapsible>
</Callout>

<Callout type="note">
<Collapsible>
[**Polynomial Commitment.**](commitment) The "lowermost" layer of our cryptographic stack deals with the assessment of evaluation claims about multilinear polynomials.

We don't say a ton here. Mostly, we refer out to [[DP24](https://eprint.iacr.org/2024/504)], which describes in detail the polynomial commitment scheme that we use.
</Collapsible>
</Callout>
