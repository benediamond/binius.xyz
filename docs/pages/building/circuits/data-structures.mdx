# Data Structures

Circuits cannot create dynamic data structures. All allocation happens at circuit construction. Operations like creating vectors, pushing elements, or extending containers at runtime don't exist in circuits:

```rust
let mut message = Vec::new();     // Can't create dynamic containers
message.push(byte);               // Can't append at runtime
message.extend(data);             // Can't concatenate dynamically
```

## Example: Concatenation

Since circuits can't dynamically join arrays at runtime, concatenation requires a different approach. Instead of building the result incrementally, the circuit pre-allocates space for the final combined array and uses constraints to verify that it contains the expected data. The circuit doesn't actually perform the concatenation - it just checks that the prover has provided the correct concatenated result:

```rust
let builder = CircuitBuilder::new();

// Two 4-element arrays
let data_a: [_; 4] = core::array::from_fn(|_| builder.add_witness());
let data_b: [_; 4] = core::array::from_fn(|_| builder.add_witness());

// Pre-allocated joined array
let joined: [_; 8] = core::array::from_fn(|_| builder.add_witness());

// Verify joined = data_a || data_b
for i in 0..4 {
    builder.assert_eq(format!("concat_a[{i}]"), joined[i], data_a[i]);
    builder.assert_eq(format!("concat_b[{i}]"), joined[i + 4], data_b[i]);
}
```

The constraints verify that `joined` contains exactly `data_a` followed by `data_b`.

For more complex scenarios involving variable-length concatenation, the framework provides `binius_frontend::circuits::concat::Concat` which handles concatenating multiple byte sequences of varying lengths. This requires additional constraint patterns to verify correct positioning and boundaries.
