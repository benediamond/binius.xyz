# Building [Everything you need to know to build end-to-end ZK applications with Binius64]

In this site area, we explain how to build real applications in Binius64.

Run our SHA-256 example by pasting the below code:

```bash
git clone https://github.com/IrreducibleOSS/binius64
cd binius64

RUSTFLAGS="-C target-cpu=native" cargo run --release --example sha256
```

We've broken up this guide into two main sections.

<Callout type="note">
<Collapsible>
[**End-to-End Example.**](example) Here, we explain the overall, end-to-end flow of a Binius64 application, from setting up a project, to building a circuit, to generating and verifying proofs about it.

We moreover show how each of these steps gets carried out in code, by dumping runnable snippets. Throughout, we focus on a classic, important cryptographic task (proving _knowledge-of-opening_ with respect to a simple, SHA-256-based commitment scheme).
</Collapsible>
</Callout>

<Callout type="note">
<Collapsible>
[**Circuits.**](circuits) Here, we dive more deeply into actually writing circuit code. We explain various tips and motifs that show up repeatedly in real applications.

We focus on recurring patterns like [branchless control flow](/building/circuits/control-flow) and [circuit composition](/building/circuits/composition). We also describe some [practical considerations](/building/circuits/performance) around performance.
</Collapsible>
</Callout>

<Callout type="note">
<Collapsible>
[**Tutorial Examples.**](https://github.com/IrreducibleOSS/binius64) All code snippets in this guide have corresponding runnable examples in the repository. To use them, first clone the repository:


Look for **"Try it"** blocks throughout the documentation to run specific examples. The `prover/examples/examples/tutorials/` directory contains additional examples beyond those covered here, including basics demonstrations and advanced circuit patterns.

</Callout>
