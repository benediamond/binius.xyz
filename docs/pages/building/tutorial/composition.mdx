# Circuit Composition [Building modular circuits]

This section covers conventions for building reusable circuit components. These conventions provide modularity and clear interfaces between circuit components, allowing you to build applications from circuits.

## Naming and Structure

Circuit building functions follow these conventions:

- Name: `circuit_build_<operation>`
- No return value
- Parameters in specific order
- Don't populate witness values

```rust
fn circuit_build_<operation>(
    builder: &CircuitBuilder,
    config_params...,    // Lengths, sizes, constants
    input_wires...,      // Wires caller provides
    output_wires...,     // Wires caller reads
) {
    // Internal wires created here
    let internal_state = builder.add_internal();
    // Build constraints only - no witness population
}
```

## Wire Management

### Parameter Wires
Pass wires that the caller needs to interact with:
- Input values to provide
- Output values to read
- Connections to other circuits

The function doesn't populate these wires - the caller is responsible for setting witness values.

### Internal Wires
Create computation wires inside the function for intermediate values. The caller doesn't see these.

### Circuit Composition
Functions can compose other `circuit_build_` functions internally. Wire interfaces are propagated through parameter passing - intermediate wires from composed circuits are only exposed if needed by the caller.

```rust
fn circuit_build_hash_twice(
    builder: &CircuitBuilder,
    input: Wire,
    output: Wire,
) {
    // Internal intermediate wire - not exposed to caller
    let intermediate = builder.add_internal();
    
    // Compose two hash circuits
    circuit_build_hash(builder, input, intermediate);
    circuit_build_hash(builder, intermediate, output);
}

fn circuit_build_hash_chain(
    builder: &CircuitBuilder,
    input: Wire,
    intermediate: Wire,  // Caller wants access to intermediate value
    output: Wire,
) {
    // Intermediate wire exposed through interface
    circuit_build_hash(builder, input, intermediate);
    circuit_build_hash(builder, intermediate, output);
}
```

## Parameter Order

1. Builder reference
2. Configuration (sizes, lengths)
3. Input wires
4. Output wires
5. Working space (if needed)

## Hierarchical Composition

```rust
fn circuit_build_merkle_node(
    builder: &CircuitBuilder,
    left: [Wire; 4],
    right: [Wire; 4],
    parent: [Wire; 4],
) {
    // Internal concatenation buffer
    let concat: Vec<Wire> = (0..8)
        .map(|_| builder.add_internal())
        .collect();
    
    // Copy children to concatenation buffer
    for i in 0..4 {
        builder.assert_eq(format!("left_{}", i), left[i], concat[i]);
        builder.assert_eq(format!("right_{}", i), right[i], concat[i + 4]);
    }
    
    // Delegate to hash circuit
    circuit_build_hash(builder, &concat, parent);
}
```
