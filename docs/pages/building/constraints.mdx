# Constraints and Shifted Indices [The Binius64 constraint system]

This section covers how constraints are structured in Binius64 and how shifted value indices enable efficient bit manipulation.

## Constraint Forms in Binius64

The formal structure of constraints in Binius64 must be understood before examining shifts.

### The Two Constraint Types

Binius64 has exactly two types of constraints:

**AND Constraint:**
```
(A_0 XOR A_1 XOR ... XOR A_n) AND (B_0 XOR B_1 XOR ... XOR B_m) = C_0 XOR C_1 XOR ... XOR C_k
```

**MUL Constraint:**
```
(A_0 XOR A_1 XOR ... XOR A_n) * (B_0 XOR B_1 XOR ... XOR B_m) =
    (HI_0 XOR HI_1 XOR ... XOR HI_p) * 2^64 + (LO_0 XOR LO_1 XOR ... XOR LO_q)
```

Where each term (A_0, B_0, etc.) can be:
- A wire reference: `v_i` (value at index i in the witness vector)
- A constant: any 64-bit value

Each operand is a XOR combination of these terms. Since XOR is addition in GF(2^64), these combinations are "free" - they don't require additional constraints. This flexibility becomes more useful when terms are extended to include shifted values.

### Cost Model

The relative costs of constraints in Binius64:

| Constraint Type | Relative Cost | Usage |
|-----------------|---------------|-------|
| AND constraint | 1x (baseline) | Bitwise operations, masking |
| MUL constraint | approx 200x | Integer multiplication |
| XOR operations | Free | Within constraint operands |
| Shifts | Free | Via shifted value indices |

## Understanding Shifted Value Indices

### The Problem: Bit Manipulation Efficiency

Consider implementing a bit rotation:
```rust
// Rotate left by 13 bits
let rotated = (value &lt;&lt; 13) \| (value &gt;&gt; 51);
```

In a bit-level circuit (the traditional approach), this rotation requires:
- 64 constraints to decompose the word to bits
- 64 constraints to rearrange the bits
- 64 constraints to recombine into a word

Total: ~192 constraints for one rotation!

### The Solution: Shifted Value Indices

Binius64 extends the constraint terms by encoding shifts directly into value references.

Each term in a constraint can be a **shifted value index** - a tuple `(value_id, shift_op, shift_amount)`.

Examples:
- `(v0, sll, 13)` means "value 0 shifted left by 13"
- `(v1, srl, 5)` means "value 1 shifted right by 5"
- `(v2, sra, 8)` means "value 2 arithmetic shift right by 8"

This means our constraint operands can now freely mix regular and shifted references:
```
(v0, sll, 13) XOR (v0, srl, 51) XOR v1 XOR 0xFF
```
All within a single operand, still "free" with no additional constraints.

### How It Works

By allowing shifted references, bit manipulations can be expressed **for free** within constraints.

Example - Rotate left by 13:
```
// Traditional: needs multiple constraints
t1 = v0 &lt;&lt; 13
t2 = v0 &gt;&gt; 51
result = t1 \| t2

// Binius64: single constraint (XOR is free)
result = (v0 sll 13) XOR (v0 srl 51)
```

### Practical Examples

#### SHA-256 Sigma Function
SHA-256's sigma_0 function: `ROTR(x,2) XOR ROTR(x,13) XOR ROTR(x,22)`

```
// Bit-level: ~192 constraints (3 rotations * 64 bits)

// Binius64: XOR shifted values (free)
s0 = (x srl 2) XOR (x sll 62) XOR
     (x srl 13) XOR (x sll 51) XOR
     (x srl 22) XOR (x sll 42)
```

Cost: 0 constraints (just XOR of shifted indices)

#### Bit Extraction
Extract bit N from value V:

```rust
// Traditional: 64 constraints to decompose to bits

// Binius64: Using CircuitBuilder API
let shifted = builder.shr(V, N);           // Shift right by N (free - shifted index)
let bit_n = builder.band(shifted, one);    // AND with 1 (one AND constraint)
```

Cost: 1 AND constraint total

### Shift Types Supported

Binius64 supports three shift operations:

1. **Logical Left Shift (sll)**: Shifts bits left, filling with zeros
2. **Logical Right Shift (srl)**: Shifts bits right, filling with zeros
3. **Arithmetic Right Shift (sra)**: Shifts bits right, preserving sign bit

Each can shift by 0-63 bits and is encoded directly in the shifted value index.

## Practical Impact

The combination of 64-bit word operations and shifted value indices provides significant efficiency gains:

| Operation | Bit-level Circuits | Binius64 | Improvement |
|-----------|-------------------|----------|-------------|
| 64-bit rotate | approx 192 constraints | 0 constraints | Infinite |
| SHA-256 round | approx 800 constraints | approx 50 constraints | 16x |
| Bit extraction | 64 constraints | 1 constraint | 64x |
| 64-bit addition | approx 320 constraints | 2 constraints | 160x |

## Value Types and Interpretations

In Binius64 circuits, 64-bit values can be interpreted in different ways depending on the operation:

### The Three Interpretations

1. **Unsigned Integers (0 to 2^64-1)**
   - Used by: `iadd_cin_cout`, `imul`, `icmp_ult`, `icmp_eq`
   - Standard arithmetic operations
   - What you typically think of as "numbers"

2. **Bit Patterns**
   - Used by: `band`, `bor`, `bxor`, shifts, rotations
   - Pure bitwise operations
   - No arithmetic meaning, just patterns of 1s and 0s

3. **Binary Field Elements (GF(2^64))**
   - Used internally by the proof system
   - Hidden from circuit writers
   - Addition is XOR, multiplication uses irreducible polynomials

### The MUL Constraint: Integer Multiplication

The MUL constraint performs **integer multiplication**, not field multiplication:

```
A * B = (HI &lt;&lt; 64) \| LO
```

This gives the full 128-bit product split into high and low words, needed for:
- RSA operations (modular exponentiation)
- Elliptic curve arithmetic
- General bignum computations

**Important**: Don't confuse this with field multiplication used internally by the proof system. The MUL constraint does regular "school math" multiplication that produces carries and can overflow.

### Practical Usage

Most circuits work with integers and bit patterns:

```rust
// Integer operations
let (sum, carry) = builder.iadd_cin_cout(a, b, cin);    // Treats as integers
let (hi, lo) = builder.imul(x, y);                      // Integer multiplication

// Bit pattern operations  
let masked = builder.band(value, mask);                 // Treats as bit patterns
let shifted = builder.shl(value, 5);                    // Bit manipulation
let rotated = builder.rotl(value, 13);                  // Bit rearrangement

// The field arithmetic happens behind the scenes in the proof system
```
