# CLI Framework

The `binius_examples::Cli` framework is a helper that provides built-in commands for proving, verification, and statistics.

```rust
use anyhow::Result;
use binius_examples::{Cli, ExampleCircuit};
use binius_frontend::compiler::{CircuitBuilder, circuit::WitnessFiller, Wire};
use clap::Args;

fn main() -> Result<()> {
    let _tracing_guard = tracing_profile::init_tracing()?;

    Cli::<YourCircuit>::new("circuit-name")
        .about("Circuit description")
        .run()
}
```

## Example: Hash Preimage Prover

To build a custom CLI, define your circuit structure with wire references, create parameter and instance types with `#[derive(Args)]` for CLI arguments, and implement the `ExampleCircuit` trait:

```rust
/// Circuit state storing wire references
struct HashPreimageExample {
    secret_wire: Wire,
    public_hash_wire: Wire,
}

/// Circuit parameters (compile-time configuration)
#[derive(Args, Clone)]
struct HashPreimageParams {
    #[arg(long, default_value = "3735928559")]  // 0xDEADBEEF
    hash_constant: u64,
}

/// Instance data (runtime values for witness)
#[derive(Args, Clone)]
struct HashPreimageInstance {
    #[arg(long, default_value = "42")]
    secret: u64,
}

impl ExampleCircuit for HashPreimageExample {
    type Params = HashPreimageParams;
    type Instance = HashPreimageInstance;

    fn build(params: Self::Params, builder: &mut CircuitBuilder) -> Result<Self> {
        let hash_constant = builder.add_constant_64(params.hash_constant);
        let secret_wire = builder.add_witness();
        let hash_result = builder.bxor(secret_wire, hash_constant);
        let public_hash_wire = builder.add_inout();
        builder.assert_eq("hash_verification", hash_result, public_hash_wire);

        Ok(Self { secret_wire, public_hash_wire })
    }

    fn populate_witness(&self, instance: Self::Instance, filler: &mut WitnessFiller) -> Result<()> {
        filler[self.secret_wire] = Word(instance.secret);
        let expected_hash = instance.secret ^ 0xDEADBEEF;
        filler[self.public_hash_wire] = Word(expected_hash);
        Ok(())
    }
}

fn main() -> Result<()> {
    Cli::<HashPreimageExample>::new("hash-preimage")
        .about("Prove knowledge of a hash preimage")
        .run()
}
```

## CLI Commands

The framework provides several built-in commands:

**Prove (default)**: Generate and verify a proof
```bash
./target/release/cli_framework prove --secret 123
```

**Statistics**: Display circuit statistics  
```bash
./target/release/cli_framework stat

# Example output:
# Circuit statistics:
#   AND constraints: 2
#   MUL constraints: 0
#   Total witness wires: 1
#   Public input/output wires: 1
#   Constraint system size: 2
```

**Composition**: Output circuit structure as JSON
```bash
./target/release/cli_framework composition
```

**Snapshot testing**: Track statistics to catch regressions
```bash
./target/release/cli_framework check-snapshot  # Verify against baseline
./target/release/cli_framework bless-snapshot   # Update baseline
```

Snapshots in `crates/examples/snapshots/` track constraint counts and circuit composition, catching unintended changes like constraint count doubling during refactoring.
