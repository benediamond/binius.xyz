# Proving Workflow

Constraint verification checks whether witness values satisfy circuit equations - essentially evaluating each constraint to confirm it equals zero. This confirms circuit logic correctness without cryptographic overhead.

Proof generation transforms this local verification into a succinct proof. The prover encodes the witness as a multilinear polynomial, commits to it cryptographically, then engages in an interactive protocol (made non-interactive via Fiat-Shamir). Through multiple rounds of sumcheck and polynomial evaluations over extension fields, the protocol produces a proof that convinces any verifier that valid witness values exist - without revealing them.

The cryptographic machinery only matters when convincing third parties. During development, use `verify_constraints` to quickly test circuit logic:


## Proving Workflow

The following example demonstrates the complete workflow for generating and verifying proofs. It builds a simple multiplication circuit, sets up the cryptographic components, generates a witness, creates a proof, and verifies it:

```rust
// 1. Build circuit
let builder = CircuitBuilder::new();
let x = builder.add_witness();           // Private input
let three = builder.add_constant_64(3);
let (_hi, lo) = builder.imul(x, three);  // x * 3
let y = builder.add_inout();             // Public output
builder.assert_eq("verify_multiplication", lo, y);
let circuit = builder.build();

// 2. Setup prover and verifier
let cs = circuit.constraint_system().clone();
let log_inv_rate = 1; // Controls proof size vs. security tradeoff

let verifier = Verifier::<StdDigest, _>::setup(
    cs.clone(),
    log_inv_rate,
    StdCompression::default()
)?;
let prover = Prover::<OptimalPackedB128, _, StdDigest>::setup(
    verifier.clone(),
    ParallelCompressionAdaptor::new(StdCompression::default())
)?;

// 3. Generate witness
let mut filler = circuit.new_witness_filler();
filler[x] = Word(7);   // x = 7
filler[y] = Word(21);  // y = 21
circuit.populate_wire_witness(&mut filler)?;
let witness = filler.into_value_vec();

// 4. Generate proof
let challenger = StdChallenger::default();
let mut prover_transcript = ProverTranscript::new(challenger.clone());
prover.prove(witness.clone(), &mut prover_transcript)?;
let proof = prover_transcript.finalize();

// 5. Verify proof
let mut verifier_transcript = VerifierTranscript::new(challenger, proof);
verifier.verify(witness.public(), &mut verifier_transcript)?;
verifier_transcript.finalize()?;
```

## Security Parameters

```rust
let log_inv_rate = 1;
```

The prover and verifier setup uses default parameters (such as `log_inv_rate`) that balance proof size and generation time. The framework handles the cryptographic security parameters internally. For production applications, the default configuration provides appropriate security guarantees. For advanced users, see [Blueprint: Polynomial Commitment](../../blueprint/commitment) for technical details about the polynomial commitment scheme.
